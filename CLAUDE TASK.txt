Нижче наведено сувору специфікацію для імплементації системи Level Up, процедурної генерації карток прокачки (Trade-offs) та UI для гри-гризні (Vampire Survivors/Brotato-like). Використовуй ці правила для створення класів, структур та логіки.
1. Характеристики Гравця (Player Stats) та Пасивка

Створити єдиний клас/структуру PlayerStats для зберігання поточних значень.

    BaseDamage (float): Базова шкода від пострілу.

    FireRateDelay (float): Затримка між пострілами в секундах (менше = швидше).

    PierceCount (int): Кількість ворогів, яких снаряд пробиває наскрізь (базово 0).

    AttackRange (float): Радіус виявлення ворогів для автострільби.

    MoveSpeed (float): Швидкість пересування гравця.

    MaxBurnout (float): Максимальний запас здоров'я (Вигорання).

    Пасивна здібність: "Аура Гумової Качечки" (Rubber Duck Aura) ПЕрероби потчний пасивний скіл на це.

        AuraRange (float): Радіус зони навколо гравця.

        AuraSlowdown (float): Множник швидкості ворогів у зоні (наприклад, 0.7 = ворог рухається на 30% повільніше).

        AuraDPS (float): Шкода, яка наноситься ворогам щосекунди, поки вони в аурі.

2. Система Досвіду (XP System) НА ЮІ ВНИЗУ ЕКРАНУ МАЄ БУТИ ШКАЛА ЗОЛОТА, КОТРА БУДЕ ЗАПОВНЮВАТИСЬ І ПОКАЗУВАТИ СКІЛЬКИ ЗАЛИШИЛОСЬ ДО НАСТУПНОГО РІВНЯ!!!

Логіка накопичення досвіду має бути лінійною і не ускладнюватися з рівнями.

    CurrentXP (float): Поточний досвід.

    MaxXP (float): Досвід, необхідний для рівня. Константа! (наприклад, завжди 100).

    Дроп досвіду з ворогів: * Звичайний баг: 10 XP

        Швидкий баг: 5 XP

        Танк-баг: 50 XP

        Турелька: 20 XP

    Логіка: Коли CurrentXP >= MaxXP, викликається подія OnLevelUp, а залишок досвіду переноситься (CurrentXP = CurrentXP - MaxXP).

3. Генератор Карток "Трейд-офів" (Trade-off Generator)

При виклику OnLevelUp система генерує 3 унікальні картки. Картка складається з Позитивного бафу (для Гравця) та Негативного бафу (для Ворогів/Гри).

А. Рідкісність (Rarity) та Ваги спавну:

    Junior (Біла): Шанс 60%. (Малий баф гравцю / Малий баф ворогам)

    Middle (Синя): Шанс 30%. (Середній баф гравцю / Відчутний баф ворогам)

    Senior (Фіолетова): Шанс 10%. (Критичний баф гравцю / Жорсткий баф ворогам)

Б. Пули позитивних ефектів (Player Buffs):
Система випадково обирає 1 стат для покращення, ось приклад. Але взагалі рандомно має будь який стат змінювати, що буде в конфігу гравця врахуй це уважно:

    Damage: Junior (+10%), Middle (+20%), Senior (+40%)

    FireRate: Junior (-5% delay), Middle (-10% delay), Senior (-25% delay)

    MoveSpeed: Junior (+5%), Middle (+10%), Senior (+20%)

    Pierce: Junior (+0), Middle (+1), Senior (+2) (особливе правило, тільки цілі числа)

    AuraRange або AuraDPS: Junior (+10%), Middle (+20%), Senior (+35%)

В. Пули негативних ефектів (Enemy Buffs):
Система випадково обирає 1 стат для погіршення ситуації на полі:
Система випадково обирає 1 стат для погіршення, ось приклад. Але взагалі рандомно має будь який стат змінювати, що буде в конфігу ворога врахуй це уважно
    GlobalEnemyHP: Всі вороги стають товстішими на +5% (Junior) / +10% (Middle) / +20% (Senior).

    GlobalEnemySpeed: Всі вороги швидшають на +2% (Junior) / +5% (Middle) / +12% (Senior).

    SpawnRateDelay: Час між спавном ворогів зменшується на -3% (Junior) / -7% (Middle) / -15% (Senior).

4. UI/UX Флоу (Поведінка інтерфейсу)

Соковитість та твіни — критично важливі для відчуття гри.

    Тригер: Спрацьовує подія OnLevelUp.

    Пауза: Встановити TimeScale = 0. Ігровий процес повністю зупиняється.

    Затемнення: На екрані плавно з'являється напівпрозора чорна підкладка (Alpha від 0 до 0.7 за 0.2 сек).

    Анімація Карток: 3 UI-картки з'являються по центру екрана.

        Твін: Scale від (0, 0, 0) до (1, 1, 1) за 0.3 сек з ефектом Overshoot (Bounce), щоб вони ніби "вистрибнули".

    Наповнення Картки (Візуал):

        Фон картки: Колір залежить від Rarity (Білий, Синій, Фіолетовий).

        Заголовок: Рандомний жарт (напр., "Junior's Quick Fix", "Senior's Refactoring").

        Позитивний текст: Зеленим кольором (напр., "+20% Швидкість руху"). текст має формуватись на основі вибраного стату рандомом

        Негативний текст: Червоним кольором (напр., "Але баги стають на 10% швидшими!"). текст має формуватись на основі вибраного стату рандомом

    Вибір (Hover та Click):

        При наведенні мишки (Hover): Картка трохи збільшується (Scale до 1.1) і підсвічується.

        При кліку (Click):

            Застосувати вибрані модифікатори до PlayerStats та EnemyGlobalConfigs.

            Зробити Scale всіх 3 карток у 0 за 0.2 сек.

            Прибрати затемнення екрана.

            Встановити TimeScale = 1 (Зняти паузу).
